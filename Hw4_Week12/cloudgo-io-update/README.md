在上次作业 cloudgo-io 的基础上修改。  
为了编写易于阅读、扩展和维护的程序，我们沿着 java jdbc 编程风格一路学到黑！  
采用 java 经典的 “entity - dao - service” 层次结构模型：  
将cloudgo-io的处理数据、操作数据库的model部分改成了entity-dao-service结构模型。  
```
+---------------+------------------+
| xxx-service   |提供原子交易服务   |
+---------------+------------------+
| xxx-dao       |提供xxx的数据存取  |
+---------------+------------------+
| xxx-entity    |定义xxx的实体数据  |
+---------------+------------------+  
```
